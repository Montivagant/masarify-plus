Faster Expense Logging: How to Reduce Taps?

Current flow: FAB → Type → Amount → Category → Wallet → Date/Note → Save (6+ taps). Target: 2-3 taps for common expenses.

Current Flow (6+ interactions)

Current: AddTransactionScreen
Step 1: Tap FAB
Step 2: Select type (Expense/Income) chips
Step 3: Enter amount
Step 4: Scroll + tap category chip
Step 5: Select wallet (if multiple)
Step 6: Optional date/note
Step 7: Tap Save
= 7 interactions minimum
A

"Smart Defaults" — Same Screen, Fewer Decisions

Change: Keep current AddTransaction screen but pre-fill everything intelligently:

  • Type = Expense (default, already done)
  • Category = auto-selected based on time of day + learned patterns
  • Wallet = last used wallet (already done) or default account
  • Date = today (already done)

User only needs to: Enter amount → Tap Save (2 taps)

Category and wallet are pre-filled but editable. User glances, confirms, saves.

Pros

  • No new UI — enhances existing screen
  • Smart defaults already partially built (SmartDefaultsProvider)
  • User can still customize everything

Cons

  • Wrong auto-category = user must scroll and fix
  • Screen still feels "heavy" visually
B

"Quick Entry Sheet" — Bottom Sheet for Fast Logging

Pros

  • Feels instant — bottom sheet stays in context
  • Most common categories visible as chips
  • Expandable for power users who want details
  • 2-tap minimum (amount → save)

Cons

  • New UI component to build
  • Limited space for many categories
  • Full screen still needed for income/transfer
C

"Hybrid" — Smart Defaults + Streamlined Screen

Change: Keep full AddTransaction screen but restructure layout:

  • Amount input at TOP (largest, auto-focused with keyboard open)
  • Frequent categories as horizontal scrollable chips (sorted by usage frequency — already built!)
  • Wallet selector only shown if user has 2+ wallets
  • Title/note/date collapsed by default (expandable section)
  • Save button sticky at bottom, always visible
  • Smart defaults pre-fill category + wallet

User flow: FAB → keyboard opens → type amount → glance at pre-selected category → Save

Pros

  • Enhances existing screen (minimal new code)
  • Keyboard auto-opens = immediate data entry
  • Collapsed optional fields reduce visual noise
  • Frequency-sorted categories already implemented

Cons

  • Still a full-screen transition (not a sheet)
  • Users must discover the expandable section

Recommendation: Option C (Hybrid)

Option C gives the best of both worlds — it reuses your existing AddTransaction screen (less code), leverages frequency-sorted chips you already built, and auto-focuses the keyboard for instant entry. The collapsed optional fields make the screen feel light for quick entries but expandable for detailed ones. No new bottom sheet component needed.
