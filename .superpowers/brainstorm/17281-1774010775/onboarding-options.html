Onboarding Redesign: How Should New Users Start?

Goal: Get users to their first transaction in under 60 seconds. No demo dashboard.

A

"Skip to Action" — Minimal Onboarding

Flow: Welcome screen (2 sec) → Language picker → Dashboard immediately

Auto-creates default wallet (Cash) + default categories. No wallet setup. User starts logging right away.

Pros

  • Fastest to first transaction (~15 sec)
  • Zero friction — nothing to configure
  • Matches Duolingo/TikTok-style instant gratification

Cons

  • Users may not understand what the app does
  • Default "Cash" wallet might confuse bank users
  • No "wow" moment
B

"Quick Setup + Coach Marks" — Guided but Fast

Flow: Welcome (language built-in) → "What's your main account?" (Bank / Cash / Mobile Wallet — single tap) → Dashboard with coach marks

2 screens max. Pick an account type, we auto-name it + set balance 0. Coach marks (tooltips) on first visit teach: FAB for expenses, long-press for voice, quick add zone.

Pros

  • Fast (~30 sec) but gives users a choice
  • Coach marks teach features organically
  • Wallet type selection feels personal

Cons

  • Coach marks can annoy power users
  • Still an "empty" dashboard initially
C

"Value Preview + Quick Setup" — Show Then Do

Flow: Welcome (language) → 3 animated slides ("Track in 2 taps" / "Voice: just say it" / "SMS auto-detect") → "Main account?" → Dashboard

Full-bleed animations showing actual app features in action. Auto-advances with skip button. ~5 sec per slide.

Pros

  • Users understand value before starting
  • Animations create "wow" moment
  • Still fast (~45 sec total)

Cons

  • Animations need Lottie design work
  • 3 slides feels like current onboarding
  • Users may skip anyway

Recommendation: Option B

Option B is the sweet spot — fast setup + organic feature teaching via dismissable coach marks. Avoids the "another carousel" feeling (C) while giving more context than raw drop-in (A).
