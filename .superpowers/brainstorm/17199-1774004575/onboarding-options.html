Onboarding Redesign: How Should New Users Start?

Goal: Get users to their first transaction in under 60 seconds. No demo dashboard.

A

"Skip to Action" — Minimal Onboarding

Flow: Welcome screen (2 sec auto-advance) → Language picker → Dashboard immediately

Auto-creates default wallet (Cash) + default categories. No wallet setup step. User starts logging right away. Settings like currency and wallet name can be changed later in Settings.

Pros

  • Fastest to first transaction (~15 sec)
  • Zero friction — nothing to configure
  • Matches how Duolingo/TikTok onboard (instant gratification)

Cons

  • Users may not understand what the app does
  • Default "Cash" wallet might confuse bank users
  • No "wow" moment — just drops into empty dashboard
B

"Quick Setup + Coach Marks" — Guided but Fast

Flow: Welcome (language picker built-in) → "What's your main account?" (Bank/Cash/Mobile Wallet — single tap) → Dashboard with coach marks highlighting FAB, voice, and quick add

2 screens max. The "main account" question replaces the full wallet form — just pick a type, we name it and set balance to 0. Coach marks (tooltips) appear on first visit to teach key features.

Pros

  • Fast (~30 sec) but gives users a choice
  • Coach marks teach features organically
  • Wallet type selection feels personal

Cons

  • Coach marks can annoy power users
  • Still an "empty" dashboard on first load
C

"Value Preview + Quick Setup" — Show Then Do

Flow: Welcome (language) → 3-slide value preview (animated: "Track in 2 taps" / "Voice: just say it" / "SMS auto-detect") → "What's your main account?" → Dashboard

The 3 slides are full-bleed animations (not text cards). Each shows the actual app feature in action — a mini demo that takes 5 seconds per slide. Auto-advances with skip button.

Pros

  • Users understand value before starting
  • Animations create "wow" moment
  • Still fast (~45 sec total)
  • Skip button for returning/impatient users

Cons

  • Animations need design/Lottie work
  • 3 slides is borderline — could feel like current onboarding
  • Users may skip and miss the value pitch

💡 Recommendation: Option B

Option B hits the sweet spot — fast setup + organic feature teaching via coach marks. It avoids the "another onboarding carousel" problem (Option C) while giving more context than a raw drop-in (Option A). Coach marks can be dismissed and never shown again.
