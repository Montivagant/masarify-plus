<task id="05-04-01" title="Remove ensureTrialStarted() from main.dart">
<read_first>
- lib/main.dart (line 87: `unawaited(subService.ensureTrialStarted());` — this is the line to remove)
</read_first>

<action>
In `main.dart`, delete line 87:
```dart
unawaited(subService.ensureTrialStarted());
```

This line currently calls `ensureTrialStarted()` on every app launch, which means the 7-day trial clock starts ticking the moment the app is first opened, even before the user completes onboarding. After removal, the trial will only start when the user completes onboarding (wired in task 05-04-02).

Keep line 86 (`unawaited(subService.initialize());`) — that initializes the IAP listener and must run on every launch.
</action>



3. After the success overlay dialog returns (line 117: `);`) and before `if (!mounted) return;` (line 118), add the trial snackbar. The modified section should look like:
   ```dart

     barrierDismissible: false,

   SnackHelper.showSuccess(context, context.l10n.trial_started_message);
   context.go(AppRoutes.dashboard);
   ```

The snackbar fires AFTER the overlay auto-dismisses and BEFORE navigating to the dashboard. Since `ScaffoldMessenger` is at the app level, the snackbar persists through the navigation and appears on the dashboard.

<action>
This is a verification task. Read the code and confirm the following:

1. **PaywallScreen** (lines 85, 150-179): `ref.watch(trialDaysRemainingProvider)` is called. The trial banner shows when `trialDays > 0`. The banner text uses `context.l10n.paywall_trial_banner(trialDays)`.

2. **SubscriptionScreen** (lines 22-24, 53-74): `ref.watch(trialDaysRemainingProvider)` is called. The trial info card shows when `isInTrial` is true and `!service.isPro`.

3. **trialDaysRemainingProvider**: Returns `service.trialDaysRemaining` which computes the remaining days from `_kTrialStartDate` in SharedPreferences.

4. **Pre-onboarding behavior**: Before `ensureTrialStarted()` is called (before onboarding completes), `trialDaysRemaining` returns `_trialDays` (7) because `_prefs.getString(_kTrialStartDate)` returns null. This means `hasProAccess = true` before onboarding. This is intentional and safe — the user won't hit limits during onboarding.

5. **Post-onboarding behavior**: After `ensureTrialStarted()` sets the trial start date, `trialDaysRemaining` returns `7 - daysSinceStart`. On day 0, it returns 7. On day 7, it returns 0 and `hasProAccess` becomes false.

If all of the above is correct, no code changes are needed. If any discrepancy is found (e.g., trial banner not showing, wrong day count), fix it.
</action>
