<task id="5">
<title>Fix SnackBar bottom margin to prevent mid-screen positioning</title>
<read_first>
- lib/core/constants/app_sizes.dart
- lib/shared/widgets/feedback/snack_helper.dart
</read_first>
<action>
The current `snackbarBottomMargin` is calculated as:
`bottomNavHeight (64) + fabVerticalOffset (24) + md (16) = 104`

This value of 104dp pushes SnackBars too high on smaller screens, making them appear mid-screen. The nav bar is 64dp and the FAB hovers ~24dp above it. The SnackBar needs to clear the FAB but not go higher.

In `lib/core/constants/app_sizes.dart`, change `snackbarBottomMargin` from:
```dart
static const double snackbarBottomMargin =
    bottomNavHeight + fabVerticalOffset + md;
```
to a more conservative value that just clears the nav bar + a small gap:
```dart
static const double snackbarBottomMargin =
    bottomNavHeight + md; // 64 + 16 = 80dp — clears nav bar with comfortable gap
```

This reduces the margin from 104dp to 80dp. The SnackBar will sit just above the nav bar instead of floating mid-screen.
</action>
<acceptance_criteria>
- grep -q "snackbarBottomMargin =" lib/core/constants/app_sizes.dart
- The value equals bottomNavHeight + md (80dp), not bottomNavHeight + fabVerticalOffset + md (104dp)
</acceptance_criteria>
</task>

<task id="8">
<title>Verify balance header center alignment in LTR and RTL</title>
<read_first>
- lib/features/dashboard/presentation/widgets/balance_header.dart
</read_first>
<action>
Per D-12, verify the balance display alignment. Current code at line 57 uses:
```dart
crossAxisAlignment: CrossAxisAlignment.start,
```

And the balance row (line 61-86) uses `Row` with `Expanded` for the balance text, which stretches full width.

The `CrossAxisAlignment.start` is correct for a header layout where text is left-aligned in LTR and right-aligned in RTL. The `Row` + `Expanded` pattern makes the balance text fill available width, which works in both directions.

If the balance text should be centered instead:
- Change `crossAxisAlignment: CrossAxisAlignment.start` to `CrossAxisAlignment.center`
- Wrap the balance `Text` in a `Center` widget or use `textAlign: TextAlign.center`

However, the Wise/Revolut style intentionally uses start alignment. Per D-12, the request is for "center-aligned" but this may conflict with the reference design. Keep as-is if the Wise-style left alignment is the intended design. The key RTL concern is that `CrossAxisAlignment.start` respects `Directionality` — it will right-align in RTL, which is correct.

No code change needed unless runtime testing shows misalignment. Mark as "verify at runtime".
</action>
