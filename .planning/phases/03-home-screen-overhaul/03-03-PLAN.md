Complete UX/UI revamp of the VoiceConfirmScreen — the gateway for all voice and AI-created transactions. Replace the current ~400-line implementation with a modern, glassmorphic full-screen form that makes parsed data clearly visible, easily editable, and inspires confidence in voice-parsed accuracy.

### Overview

Rewrite `VoiceConfirmScreen` as a modern glassmorphic form following decisions D-19 through D-22. Keep the existing `_EditableDraft` internal model and save/transfer logic — refactor only the UI layout and user interaction. Extract the single-draft form into a separate `DraftCard` widget.

### A. Create `lib/features/voice_input/presentation/widgets/draft_card.dart`

A `StatelessWidget` (or `ConsumerWidget` if it needs providers) for a single draft form:

**Props:**
- `draft` (_EditableDraft or equivalent mutable state)
- `categories` (List of CategoryEntity)
- `wallets` (List of WalletEntity)
- `onAmountChanged` (callback)
- `onCategoryChanged` (callback)
- `onWalletChanged` (callback)
- `onToWalletChanged` (callback — for transfers)
- `onDateChanged` (callback)
- `onNotesChanged` (callback)
- `onTypeChanged` (callback)
- `onSubscriptionSuggestionAccepted` (callback)
- `showSubscriptionSuggestion` (bool)
- `amountMissing` (bool — from draft.amountPiastres == null)

**Layout (inside a scrollable column within a GlassCard tier: card):**

1. **Type indicator row** at top: Row of small chips showing the transaction type (Expense/Income/Transfer/Cash Withdrawal/Cash Deposit). Active type chip is filled with type color. Tappable to change type.

2. **Amount section** (prominent, per D-19):
   - Large centered amount with explicit sign prefix: "- 1,500 EGP" (expense, red) / "+ 500 EGP" (income, green) / "↔ 2,000 EGP" (transfer, blue)
   - Text style: `context.textStyles.displaySmall?.copyWith(fontWeight: FontWeight.w700, color: typeColor)`
   - Where `typeColor` = `context.appTheme.expenseColor` / `incomeColor` / `transferColor`
   - Tappable to edit: opens the `AmountInput` widget or focuses an inline amount field
   - **Missing amount (D-20):** When `amountPiastres == null`:
     - Show "Amount not detected — please enter" in `context.colors.error` text
     - Amount field border/background highlighted with `context.colors.errorContainer`
     - The field auto-focuses so user can immediately type

3. **Category field** (per D-19):
   - `ListTile`-style row with: category icon (colored circle), category name, trailing chevron
   - Tappable: opens category search picker (reuse existing picker from add_transaction flow — find the picker widget used there)
   - If no category resolved: show "Select category" placeholder in `context.colors.outline`

4. **Account field** (per D-19):
   - `ListTile`-style row with: wallet icon, wallet name, trailing chevron
   - Tappable: opens wallet selector bottom sheet or dropdown
   - For transfers: show TWO account fields — "From" and "To" with a directional arrow between them
   - **RTL arrow flip (D-22):** Use `Directionality.of(context) == TextDirection.rtl` to flip the arrow. In LTR: "CIB → NBE". In RTL: "CIB ← NBE" (arrow points left). Or use `AppIcons.arrowRight` wrapped in `Transform.flip` for RTL.

5. **Date field** (per D-19):
   - Row with calendar icon + formatted date
   - Tappable: opens `AppDatePicker` or `showDatePicker()`
   - Default: today + dateOffset from draft

6. **Notes field** (per D-19):
   - Text field pre-filled with parsed title/merchant description
   - Editable, optional
   - Hint: "Add notes..." using l10n

7. **Subscription suggestion** (per D-19, only when `showSubscriptionSuggestion`):
   - A compact banner/chip: "🔄 Add to Subscriptions & Bills?" with Accept/Dismiss buttons
   - Accept: fires `onSubscriptionSuggestionAccepted` callback
   - Positioned below the notes field

Use `EdgeInsetsDirectional` everywhere for RTL safety (D-22).
Use design tokens (`AppSizes.*`, `context.colors.*`, `AppIcons.*`) exclusively — no hardcoded values.
Use `GlassCard(tier: GlassTier.inset)` for each form section group.

### B. Rewrite `lib/features/voice_input/presentation/screens/voice_confirm_screen.dart`

**Keep:** The `_EditableDraft` internal model, the save logic (both transaction and transfer save paths), the wallet resolution logic, the category resolution logic, the subscription detection check. Copy these from the existing file.

**Replace:** The entire `build()` method and all UI-building helper methods.

**New layout:**

For **single draft** (most common case):
```dart
Scaffold(
  appBar: AppAppBar(title: context.l10n.voice_confirm_title, ...),
  body: SingleChildScrollView(
    padding: EdgeInsetsDirectional.all(AppSizes.md),
    child: DraftCard(
      draft: _editableDrafts[0],
      categories: categories,
      wallets: wallets,
      // ... all callbacks wired to setState
      amountMissing: _editableDrafts[0].amountPiastres == null,
      showSubscriptionSuggestion: _isSubscription(0),
    ),
  ),
  bottomNavigationBar: SafeArea(
    child: Padding(
      padding: EdgeInsetsDirectional.all(AppSizes.md),
      child: AppButton(
        label: context.l10n.common_save,
        onPressed: _canSave() ? () => _save(0) : null, // Disabled when amount missing (D-20)
        isLoading: _saving,
      ),
    ),
  ),
)
```

For **multi-draft** (D-21 — optimize for fewest taps):
Use a `PageView` with `DraftCard` per page, plus a "Save & Next" button that saves the current draft and advances to the next page. Show a page indicator (e.g., "1/3") in the app bar or above the card. When the last draft is saved, pop the screen.

Alternative (if simpler): Stack all drafts in a scrollable list, each with its own "Save" button. Saved drafts show a green checkmark overlay. When all are saved, auto-pop.

Choose whichever approach requires fewer taps per draft. The `PageView` approach requires N taps (Save & Next per draft). The stacked list requires N taps too but shows all at once. Use `PageView` for cleaner UX — one draft at a time, full focus, no scrolling between drafts.

**Save button state (D-20):**
- Disabled (`onPressed: null`) when `_editableDrafts[currentIndex].amountPiastres == null || _editableDrafts[currentIndex].amountPiastres == 0`
- Shows loading spinner when `_saving`
