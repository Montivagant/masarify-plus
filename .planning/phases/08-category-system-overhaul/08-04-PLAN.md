**Create lib/features/categories/presentation/widgets/category_delete_flow.dart:**

This is a stateful widget that manages a 3-step dialog flow per D-05. It is shown via `showDialog()`.

Parameters:
- `categoryId: int` (required)
- `categoryName: String` (required)
- `onDeleted: VoidCallback?` (fires after successful delete for snackbar)

The widget uses `categoryImpactProvider` to load impact counts. Logic:

**Pre-check:** If all counts are 0, show 1-step confirm dialog per D-04:
- Title: `category_delete_empty_confirm` with `{name}` placeholder
- Actions: "Cancel" (text button) + "Delete" (filled, destructive color)
- On confirm: call `ref.read(categoryRepositoryProvider).hardDelete(categoryId)`, pop via `context.pop()`, fire `onDeleted`

**Step 1 - Impact Preview (when any count > 0):**
- Title: `category_delete_impact_title` with `{name}`
- Impact rows: only show rows where count > 0, each with an Icon + count text:
  - `AppIcons.transaction` + `category_delete_impact_transactions(count: n)`
  - `AppIcons.budget` or similar + `category_delete_impact_budgets(count: n)`
  - `AppIcons.subscription` or similar + `category_delete_impact_subscriptions(count: n)`
  - `AppIcons.learn` or similar + `category_delete_impact_patterns(count: n)`
- Actions: "Cancel" (text) + `category_delete_review_options` (filled, destructive)

**Step 2 - Archive Suggestion:**
- Body: `category_delete_archive_suggestion` text
- Highlighted card with warning color border showing archive option
- Actions: `category_delete_archive_action` (outlined, warning color) + `category_delete_permanent` (filled, destructive)
- Archive action: call `ref.read(categoryRepositoryProvider).archive(categoryId)`, pop via `context.pop()`, show archive success snackbar

**Step 3 - Force Migration Picker:**
- Title: `category_delete_migrate_title`
- Subtitle: `category_delete_migrate_subtitle(count: totalCount)`
- Inline grid of categories (excluding source) using GlassCategoryIcon tiles -- same layout as CategoryPickerSheet but embedded
- `category_delete_migrate_confirm` button -- DISABLED until target selected
- On confirm: call `ref.read(categoryRepositoryProvider).mergeCategories(categoryId, targetId)` which does the atomic merge+delete in one transaction. Pop via `context.pop()`, fire `onDeleted`.

Use `HapticFeedback.mediumImpact()` on delete/archive success.
Step transitions use `AnimatedSwitcher` with duration `AppDurations.animQuick` (200ms) and `easeInOut` curve.

The widget must be a `ConsumerStatefulWidget` to access Riverpod providers.

**IMPORTANT:** Use `context.pop()` (go_router) for ALL dialog dismissals -- NEVER use `Navigator.pop(context)` per CLAUDE.md rules.
