| Accent (10%) | `AppColors.primary` (#3DA37A Mint) | `AppColors.primaryDark` (#6B5B95 Purple) | See reserved-for list below |
| Destructive | `AppColors.expenseRed` (#C4384A) | `AppColors.expenseRedDark` (#B85450) | Delete swipe action background, "Delete permanently" button, destructive step 3 confirm |
| Warning | `AppColors.warning` (#B8860B) | `AppColors.warningDark` (#D4A574) | Archive suggestion highlight in delete step 2 |

### Accent reserved for (exactly these elements, nothing else):
1. Selected category highlight border in picker (`colors.primary` border, `colors.primaryContainer` fill)
2. "Add Category" FAB/button icon
3. Active search bar focus border
4. Drag handle reorder grip icon when actively dragging
5. "Most Used" section star icon

### Glass circle icon coloring:
- The glass circle uses `GlassTier.inset` (sigma 8, `glassInsetSurface` base)
- `tintColor`: category's `colorHex` at `alpha: 0.35` via `Color.withValues(alpha: 0.35)`
- Icon inside: `AppColors.white` always (white Phosphor icon on colored glass)
- Border: `showBorder: true` (standard inset border)
- Border radius: fully circular (`size / 2`)
- Device fallback: `GlassConfig.shouldBlur(context)` determines blur vs opaque; `GlassCard` handles this automatically

## Component Inventory

### New Widgets (to be created)

#### 1. `GlassCategoryIcon`
**File:** `lib/shared/widgets/category/glass_category_icon.dart`
**Purpose:** Renders a category's Phosphor icon inside a colored glass circle.

| Property | Type | Default | Notes |
|----------|------|---------|-------|
| `iconName` | `String` | required | Maps via `CategoryIconMapper.fromName()` |
| `colorHex` | `String` | required | Category color, used as tint at 0.35 alpha |
| `size` | `double` | `AppSizes.categoryChipSize` (52) | Outer circle diameter |
| `isSelected` | `bool` | `false` | Adds primary border when true |

**Rendering spec:**
- `GlassCard` with `tier: GlassTier.inset`
- `padding: EdgeInsets.zero`
- `showBorder: true`
- `tintColor: ColorUtils.fromHex(colorHex).withValues(alpha: 0.35)`
- `borderRadius: BorderRadius.circular(size / 2)`
- Child: `SizedBox(width: size, height: size)` containing centered `Icon(icon, color: AppColors.white, size: size * 0.5)`
- Selected state: wraps in additional `Container` with `Border.all(color: colors.primary, width: AppSizes.borderWidthFocus)` and `colors.primaryContainer` background

#### 2. `CategoryPickerSheet`
**File:** `lib/shared/widgets/pickers/category_picker_sheet.dart`
**Purpose:** Unified category picker replacing 3 inline implementations.

| Property | Type | Default | Notes |
|----------|------|---------|-------|
| `categories` | `List<CategoryEntity>` | required | Full non-archived list |
| `selectedId` | `int?` | `null` | Currently selected category |
| `onSelected` | `ValueChanged<int>` | required | Fires on tap |

**Layout spec:**
- `DraggableScrollableSheet` with `initialChildSize: AppSizes.sheetInitialSize` (0.6), `minChildSize: AppSizes.sheetSmallInitialSize` (0.4), `maxChildSize: AppSizes.sheetMaxSize` (0.85)
- Glass card background using `GlassTier.background` for the sheet container
- **Header:** Drag handle + "Select Category" title (`headlineSmall` at 20sp, SemiBold)
- **Search bar:** `TextField` with `AppIcons.search` prefix, `borderRadiusFull` (100), `contentPadding: horizontal md, vertical sm`
- **"Most Used" section:** Top 5 categories by `usageCount DESC` where `usageCount > 0`. Section header: "Most Used" with star icon in accent color. 4-column grid. Only shown when not searching.
- **"All Categories" section:** Remaining categories grouped by type (Expense header, then Income header). 4-column grid. `crossAxisCount: 4`, `childAspectRatio: 0.85`, `crossAxisSpacing: AppSizes.xs`, `mainAxisSpacing: AppSizes.xs`.
- **Grid tile:** `GlassCategoryIcon` (52px) + 4px gap + category name (`bodySmall` at 12sp, centered, `maxLines: 1`, `overflow: TextOverflow.ellipsis`)
- **Search filter:** Matches `name`, `nameAr`, and `iconName` (case-insensitive). When searching, flat list replaces sections.
- **Empty search:** "No categories found" text (`bodySmall`, outline color, centered)

#### 3. `CategoryDeleteFlow`
**File:** `lib/features/categories/presentation/widgets/category_delete_flow.dart`
**Purpose:** 3-step progressive deletion dialog.

**Step 1 --- Impact Preview:**
- Dialog with `headlineSmall` title (20sp): "Delete {categoryName}?"
- Impact rows using `Icon` + count text:
  - Transactions icon + "{n} transactions"
  - Budgets icon + "{n} budgets"
  - Subscriptions icon + "{n} subscriptions"
  - Learning patterns icon + "{n} learned patterns"
- Only show rows where count > 0
- Bottom: "Cancel" (text button) + "Review Options" (filled button, destructive color)
- If ALL counts are zero: skip to 1-step confirm per D-04

**Step 2 --- Archive Suggestion:**
- Dialog body: "Archiving hides this category without losing data. You can restore it later."
- Highlighted card with warning color border: "Archive instead?" with archive icon
- Bottom: "Archive" (outlined button, warning color) + "Delete Permanently" (filled button, destructive)

**Step 3 --- Force Migration Picker:**
- Title: "Move data to..."
- Subtitle: "All {totalCount} items will be moved to the selected category."
- Category picker grid (same layout as `CategoryPickerSheet` but inline, excluding the source category)
- "Confirm Migration" button (filled, primary color) --- disabled until a target is selected
- This step CANNOT be skipped; user MUST pick a migration target

#### 4. `CategoryMergeSheet`
**File:** `lib/features/categories/presentation/widgets/category_merge_sheet.dart`
**Purpose:** First-class "Merge into..." action.

- `DraggableScrollableSheet` with `initialChildSize: 0.5`, `maxChildSize: 0.85`
- Title: "Merge {sourceName} into..."
- Impact preview: "{n} transactions, {n} budgets, {n} subscriptions will be moved"
- Category picker grid (excluding source category)
- "Merge Categories" button (filled, primary) --- disabled until target selected
- Confirmation snackbar after success: "{sourceName} merged into {targetName}"

### Modified Widgets

#### 5. `CategoriesScreen` (Management Screen Overhaul)
**File:** `lib/features/categories/presentation/screens/categories_screen.dart`
**Modification type:** Incremental --- add swipe actions, usage badges, drag-drop, remove groupType/isDefault UI

**Focal point:** The primary visual focus of this screen is the category list tile --- each tile's `GlassCategoryIcon` (40px) paired with the category name and usage badge communicates at a glance which categories are active and how frequently they are used. The swipe actions provide secondary discovery through interaction.

**Layout spec (post-modification):**
- AppBar: "Categories" title + Add button (existing)
- Body: `ListView` with two sections (Expense / Income), each with:
  - `_SectionHeader`: type label + count badge (existing pattern)
  - Category tiles using `GlassCard` with `showShadow: true`
- **Each tile contains:**
  - Leading: `GlassCategoryIcon` (40px via `AppSizes.iconContainerMd`)
  - Title: category name (`bodyLarge`, 16sp, Regular 400)
  - Subtitle: usage count badge --- "{n} transactions" (`bodySmall`, 12sp, Regular 400, `colors.outline`)
  - Trailing: no "Default" chip, no delete icon button (replaced by swipe actions)
- **Swipe actions** (via `Slidable` + `SlidableAutoCloseBehavior`):
  - Swipe start (leading in LTR / trailing in RTL): "Merge" action with primary color icon
  - Swipe end (trailing in LTR / leading in RTL): "Archive" action (warning color) + "Delete" action (destructive color)
- **Drag-drop reorder:** `ReorderableListView` wrapping each section. Drag handle icon on long-press. Persists `displayOrder` column.
- **Remove:** `_groupLabel` helper, `groupType` subtitle, "Default" chip, `isDefault` delete guard
- **Stagger animations:** Preserved from existing `flutter_animate` pattern

#### 6. `AddCategoryScreen` (GroupType Removal)
**File:** `lib/features/categories/presentation/screens/add_category_screen.dart`
**Modification type:** Incremental --- remove groupType state, group chips UI, save parameter

**Remove:**
- `_groupType` state variable and its `setState` calls
- Group type chip row UI (needs/wants/savings selector)
- `groupType` parameter in save method call
- Corresponding l10n references in form

**Keep unchanged:** Name fields (EN + AR), icon picker, color picker, type selector (expense/income/both), save validation

## Interaction Contracts

### Category Picker (All Screens)

| Interaction | Behavior |
|-------------|----------|
| Open picker | `showModalBottomSheet` with `isScrollControlled: true`, `useSafeArea: true`, `GlassTier.background` sheet |
| Tap category | Fires `onSelected(categoryId)`, closes sheet via `Navigator.pop()` |
| Search typing | Debounce at 0ms (immediate filter, `setState` in sheet). Flat list replaces grouped sections. |
| Search clear | X icon in search field, restores grouped view |
| Empty search | Centered text "No categories found" |
| Scroll | `DraggableScrollableSheet` draggable between 40%-85% screen height |

### Category Management Screen

| Interaction | Behavior |
|-------------|----------|
| Tap category tile | Navigate to edit screen via `context.push(AppRoutes.editCategoryPath(id))` |
| Swipe-end (delete direction) | Reveal two actions: "Archive" (warning) + "Delete" (destructive). `Slidable` with `SlidableAutoCloseBehavior`. |
| Swipe-start (merge direction) | Reveal "Merge" action (primary color). Opens `CategoryMergeSheet`. |
| Long-press tile | Activates drag-drop reorder mode. Tile elevates with shadow. |
| Drag-drop release | Persists new `displayOrder` to DB. Haptic feedback (`HapticFeedback.mediumImpact()`). |
| Tap "Add" | Navigate to `AddCategoryScreen` via `context.push(AppRoutes.categoryAdd)` |

### Delete Flow

| Step | Trigger | Interaction |
|------|---------|-------------|
| Pre-check | Swipe-end "Delete" tapped | Count connected data via `categoryDao.countConnectedData(id)` |
| Branch: empty | All counts == 0 | 1-step confirm dialog. "Delete {name}? This cannot be undone." Confirm = hard delete. |
| Step 1 | Any count > 0 | Show impact preview dialog with counts. "Review Options" or "Cancel". |
| Step 2 | User taps "Review Options" | Show archive suggestion. "Archive" (safe) or "Delete Permanently" (dangerous). |
| Step 3 | User taps "Delete Permanently" | Show migration picker. MUST select a target. "Confirm Migration" triggers atomic merge+delete in Drift transaction. |
| Post-delete | Migration complete | Haptic feedback. Snackbar: "{name} deleted. {n} items moved to {targetName}." Duration: `snackbarDefault` (3s). |

### Merge Flow

| Step | Trigger | Interaction |
|------|---------|-------------|
| Open | Swipe-start "Merge" tapped | Open `CategoryMergeSheet` bottom sheet |
| Preview | Sheet opens | Show source name, impact counts. Grid of target categories (excluding source). |
| Select target | Tap a category in grid | Highlight with selected state. Enable "Merge Categories" button. |
| Confirm | Tap "Merge Categories" button | Atomic Drift transaction: migrate all data, combine usageCount, hard delete source. |
| Post-merge | Transaction complete | Close sheet. Haptic feedback. Snackbar: "{source} merged into {target}." Duration: `snackbarDefault` (3s). |

All durations use `AppDurations.*` tokens. No new duration constants introduced.

| Animation | Duration | Curve | Token |
