<objective>
Reduce default categories to ~20 essentials and remove groupType from the add/edit UI.

Purpose: Simplify the default set for new users (D-08) and remove the obsolete needs/wants/savings grouping UI that was removed from the schema in Plan 01 (D-17).
Output: Lean 20-category seed, clean add_category_screen without groupType.

NOTE on D-08 voice_dictionary.dart: Keywords in voice_dictionary.dart are intentionally KEPT -- they are NOT pruned even though some default categories are removed. Rationale: dictionary keywords map to `iconName` strings (not categoryId integers), so category delete/recreate/merge cycles never cause dictionary conflicts. Additionally, learned patterns in the `category_mappings` DB table take priority over static dictionary entries, making the dictionary a harmless fallback layer.
</objective>
